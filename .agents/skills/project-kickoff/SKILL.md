# Skill: Project Kickoff

Padrão de entrada para novas demandas — usado pelo `project-manager`.

## Quando usar
Ao receber qualquer nova tarefa ou projeto do usuário.

## Fluxo
1. Entender o objetivo de negócio (o quê e por quê)
2. Identificar se é demanda de produto ou técnica
3. Delegar ao agente correto:
   - Produto/backlog/apresentação → `product-owner`
   - Arquitetura/código/infra → `tech-lead`
   - Ambos → aciona os dois e consolida

## Perguntas-chave antes de delegar
- Qual o resultado esperado?
- Há prazo ou dependências?
- Quais agentes precisam ser envolvidos?

## Formato de saída
Resumo em 3 linhas: objetivo, delegação, próximo passo.
