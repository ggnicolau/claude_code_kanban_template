# Skill: Backlog Management

Padrão para gestão de backlog e kanban — usado pelo `product-owner`.

## Quando usar
Ao criar, priorizar ou atualizar issues no GitHub Projects.

## Estrutura de uma boa issue
- **Título**: ação + objeto (ex: "Adicionar autenticação via OAuth")
- **Contexto**: por que isso importa?
- **Critério de aceite**: lista checkável do que define "done"
- **Labels**: `bug`, `feature`, `chore`, `research`

## Priorização
- 🔴 Crítico — bloqueia outros ou está em produção quebrado
- 🟠 Alto — impacto direto no usuário ou negócio
- 🟡 Médio — melhoria relevante sem urgência
- 🔵 Baixo — backlog para quando houver espaço

## Fluxo Kanban
`Backlog` → `Todo` → `In Progress` → `In Review` → `Done`
